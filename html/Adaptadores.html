ADAPTADORES

Diseños

Todos los elementos del diseño se crear usando una jerarquía de objetos View y ViewGroup. Una View suele mostrar un elemento que el usuario puede ver y con el que puede interactuar. Por su parte, ViewGroup es un contenedor invisible que define la estructura de diseño de View y otros objetos ViewGroup

Atributos

Cada objeto View y ViewGroup admite su propia variedad de atributos XML. Algunos atributos son específicos de un objeto View (por ejemplo, TextView admite el atributo textSize), aunque estos atributos también son heredados por cualquier objeto View que pueda extender esta clase. Algunos son comunes para todos los objetos View, porque se heredan de la clase raíz View (como el atributo id).

El símbolo arroba (@) al comienzo de la string indica que el analizador de XML debe analizar y expandir el resto de la string de ID e identificarla como un recurso de ID. El símbolo más (+) significa que es un nuevo nombre de recurso que se debe crear y agregar a nuestros recursos (en el archivo R.java).


Luego, crear una instancia del objeto View y capturarla desde el diseño (generalmente en el método onCreate())


Creación de diseños con un adaptador

Cuando el contenido del diseño sea dinámico o no sea predeterminado, se puede usar un diseño con la subclase AdapterView para completar el diseño con vistas durante el tiempo de ejecución. Una subclase de la clase AdapterView usa un Adapter para enlazar datos con el diseño. Adapter se comporta como intermediario entre la fuente de datos y el diseño AdapterView; Adapter recupera los datos (de una fuente como una matriz o una consulta a la base de datos) y convierte cada entrada en una vista que puedes agregar al diseño AdapterView.

Crear una lista con RecyclerView

El widget RecyclerView es una versión avanzada y flexible de ListView.

Se utiliza cuando nuestra aplicación necesita mostrar una lista desplazable de elementos basada en grandes conjuntos de datos (datos que cambian con frecuencia).

  • En RecyclerView varios componentes diferentes trabajan juntos para mostrar sus datos.
  • El contenedor general para su interfaz de usuario es un objeto RecyclerView que agrega a su diseño.
  • Puede usar uno de los administradores de diseño estándar( LinearLayoutManager o GridLayoutManager).

    • Objetos de titular de vista

    Estos objetos son instancias de una clase definida al extender RecyclerView.ViewHolder.
  • Ejemplo: si una lista muestra una colección de música, cada titular de vista puede representar un solo álbum.

    • Si el usuario cambia la dirección en la que se desplaza, los titulares de la vista que se desplazaron fuera de la pantalla se pueden traer de vuelta. Los objetos del titular de la vista son administrados por un adaptador, que se crea al extender RecyclerView.Adapter.
